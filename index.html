📚 Understanding the Parameters
Learn what each configuration option means for your video streaming infrastructure

🎥 Number of Cameras

Total camera streams your system will handle simultaneously. Each camera requires dedicated processing power, memory, and bandwidth. Enterprise systems typically handle 64-512 cameras per server.

🎬 Frames Per Second (FPS)

Video frame rate affects processing load and bandwidth. Higher FPS provides smoother video but requires more CPU power and memory. Common values: 15fps (standard), 30fps (smooth), 60fps (high-performance).

📺 Input Resolution

Raw video resolution from cameras. Higher resolutions require exponentially more processing power and memory. 1080p is standard for most applications, 4K for high-detail requirements.

📱 Output Resolution

Transcoded resolution for viewers. Often lower than input to optimize bandwidth and compatibility. Transcoding from high input to lower output requires significant CPU resources.

🌐 Output Bitrate

Data rate for encoded video streams in Kbps. Higher bitrates provide better quality but consume more bandwidth. Typical values: 2-4 Mbps (720p), 6-10 Mbps (1080p).

💾 Storage Duration

How long recordings are kept. Affects total storage requirements. Consider legal compliance, business needs, and storage costs. Enterprise typically ranges from 30-365 days.

👥 Concurrent Viewers

Simultaneous viewers per camera stream. Each viewer requires additional egress bandwidth. Impacts network infrastructure requirements but not processing power per camera.

🔬 Technical Architecture & Calculations
Understanding how we calculate your hardware requirements

💻 CPU Requirements

1 Core per Stream: Each video stream requires dedicated processing for decoding, transcoding, and encoding. Modern CPUs can handle 1-2 streams per core depending on resolution and complexity.

🧠 RAM Calculation

Multi-layered Memory Model: Includes frame buffers (raw video data), processing overhead (200% for real-time encoding), system buffers, and network buffers. Higher resolutions exponentially increase RAM needs.

💾 Storage Formula

Bitrate × Time × Cameras: Total storage = Output bitrate × Duration in seconds × Number of cameras. Considers compression efficiency and video quality requirements.

🌐 Network Bandwidth

Ingest + Egress: Ingest bandwidth (cameras to server) + Egress bandwidth (server to viewers). Each camera typically requires 5 Mbps ingest, egress depends on viewer count and quality.

🔌 Network Interface

Bandwidth-based Selection: Recommends NIC speed based on total bandwidth requirements with overhead. Includes redundancy considerations for enterprise deployments.

⚡ Performance Factors

Real-world Considerations: Includes processing overhead, buffer management, network protocols, and system resources. Enterprise systems require 20-30% additional capacity for peak loads.
